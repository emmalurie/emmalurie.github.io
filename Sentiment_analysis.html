Using the Facebook API¶

Emma Lurie

Getting data from Facebook is not trivial, Facebook has restrictions on what data to collect and how. We'll have to start by using our own FB accounts to create an app and get its ID and secret. While usually the next step is to use these to generate a so-called OAuth token, we'll skip that step and combine appId + appSecret in the URL to serve as an identification token.

You shouldn't publish the app secret on the open Web, since others might use it to "impersonate" your app. Thus, once you have completed this notebook and displayed all results, replace the appSecret value with a dummy one.

Credits: This notebook was based on Max Woolf's "How to scrape data from Facebook" tutorial.

Eni Mustafaraj made several modifications, such as replacing the urllib2 library with the better performing requests, as well as simplifying the code to make it more "Pythonic". Some data analysis exercises were also added.

Dealing with network errors¶

When retrieving data automatically, often we will run into network errors, such as the Internal Error 500, which is a server error over which we don't have control. In that case, we want our code to be robust and keep trying.

In [369]:
import datetime

def request_until_succeed(url):
    success = False
    while success is False:
        try: 
            resp = requests.get(url)
            if resp.status_code == 200:
                success = True
        except Exception, e:
            print e
            time.sleep(5)           
            print "Error timestamp:", datetime.datetime.now()

    return resp.text

Get more data from the feed¶

The entries we're getting from the feed contain minimal information (only a few fields). We'll modify the URL to request for more fields, especially information such as number of likes, comments, and shares. For the fields of likes and comments we can also request a summary of list of users who participated in the action. By providing a value for the limit function, we can control the number of users returned for these two fields. The field shared only displays the total number of counts, not a list of users (I got an error message saying: "Subfields are not supported by shares").

In [373]:
def getFacebookPageFeedData(pageId, accessToken, numStatuses):
    "Function to retrieve a specific number of statuses from the feed, with more fields per status."
    
    # construct the URL string
    base = "https://graph.facebook.com/v2.4/{node}/feed/?access_token={token}"
    firstHalf = base.format(node=pageId, token=accessToken)
    secondHalf = "&fields=message,link,created_time,type,name,id,likes.limit(1).summary(true),comments.limit(1).summary(true),shares&limit="
    url = firstHalf + secondHalf + str(numStatuses)
    print url # to test on the browser, in case it doesn't work
    # retrieve data using the helper function that deals with connection errors
    data = json.loads(request_until_succeed(url))
    
    return data

Parsing each post¶

Each post may contain more information than we want or some posts will miss some information we expect. This function will try to "normalize" the data, so that we can put them on a table.
The code below contains a Python idiom that you might not be familiar with, involving the use of if / else in an assignment statement. That is similar to the ternary operator in Java, in case you've seen it.

In [374]:
import datetime

def processFacebookPageFeedStatus(status):
    '''returns a dictionary summarizing the data from a speciic status'''
    statusDct = {} # empty dict to store the fields
    
    statusDct['id'] = status['id']
    statusDct['message'] = status.get('message', '').encode('utf-8') # deal with unicode values
    statusDct['linkName'] = status.get('name', '').encode('utf-8')
    statusDct['statusType'] = status['type']
    statusDct['link'] = status.get('link', '')
    
    # Time needs special care since a) it's in UTC and
    # b) it's not easy to use in statistical programs.
    
    pubDate = datetime.datetime.strptime(status['created_time'],'%Y-%m-%dT%H:%M:%S+0000')
    pubDate = pubDate + datetime.timedelta(hours=-5) # EST
    pubDate = pubDate.strftime('%Y-%m-%d %H:%M:%S') # best time format for spreadsheet programs
    statusDct['datetime'] = pubDate
    
    # Nested items require chaining dictionary keys.
    # Notice the use of if/else in these statements. They mimic the "ternary" operation you might know from Java
    statusDct['likes'] = 0 if 'likes' not in status.keys() else status['likes']['summary']['total_count']
    statusDct['comments'] = 0 if 'comments' not in status.keys() else status['comments']['summary']['total_count']
    statusDct['shares'] = 0 if 'shares' not in status.keys() else status['shares']['count']
    
    return statusDct

Putting all together: collecting the entire feed¶

The following function will try to collect all posts for a given page. Facebook let's you get 100 posts at a time, thus, we'll keep sending requests until we get all posts. In order to do that, we'll exploit the fact that in every response, Facebook includes the URL for the next bunch of 100 posts.

In [375]:
def scrapeFacebookPageFeedStatus(pageId, accessToken):
    
    allStatuses = []
        
    hasNextPage = True
    numProcessed = 0   # keep a count on how many we've processed
    scrapeStartTime = datetime.datetime.now()

    print "Scraping {page} Facebook Page: {time}\n".format(page=pageId, 
                                                           time=scrapeStartTime)

    statuses = getFacebookPageFeedData(pageId, accessToken, 100)

    while hasNextPage:
        for status in statuses['data']:
            statusDct = processFacebookPageFeedStatus(status)
            allStatuses.append(statusDct)
            
            # output progress occasionally to make sure code is not stalling
            numProcessed += 1
            if numProcessed % 500 == 0:
                print "{count} Statuses Processed: {time}".format(count=numProcessed, 
                                                                  time=datetime.datetime.now())
        # if there is no next page, we're done.
        if 'paging' in statuses:
            statuses = json.loads(request_until_succeed(statuses['paging']['next']))
        else:
            hasNextPage = False


    print "\nDone!\n{count} Statuses Processed in {time}".format(count=numProcessed, 
                                                                  time=datetime.datetime.now() - scrapeStartTime)

    return allStatuses

allStatuses = scrapeFacebookPageFeedStatus(pageId, accessToken)

                    repeated_triples.append(data[i-1])

def averageDailyPolarity(triples): 
    polarities = []
    for triple in triples: 
        polarities.append(triple[1])
    return np.mean(polarities)
